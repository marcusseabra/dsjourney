ELASTICSEARCH
:: Elasticsearch is a distributed document store
:: Elasticsearch stores complex data structures that have been serialized as JSON documents
:: Stored documents are distributed across the cluster and can be accessed immediately from any node

Importante: https://www.elastic.co/pt/blog/found-crash-elasticsearch
Onde parei: https://www.elastic.co/guide/en/elasticsearch/reference/current/documents-indices.html

Conceitos Básicos
	:: Near Realtime (NRT)
	:: Cluster - Conjunto de nós (servidores) que, juntos, contêm os dados da aplicação.
		Por padrão, o nome do cluster é "elasticsearch"
		Configurar Clusters diferentes para os ambientes de desenvolvimento, homologação e produção
	:: Node - Servidor que é parte de um cluster
	:: Index (Índice) - Representam as tabelas
	:: Document - Unidade de informação que pode ser indexada. Representa os registros do índices.
	:: Shards e Replicas

Iniciando Cluster e Node
	:: ./elasticsearch - Inicia o Elasticsearch com o cluster padrão e um nó gerado randomicamente.
	:: É possível estabelecer os nomes dos nós e do cluster
		./elasticsearch -Ecluster.name=my_cluster_name -Enode.name=my_node_name
		./elasticsearch -Ecluster.name=compesa_gsc -Enode.name=seabra
		
GET /_cat/nodes?v :: Informa o nó
ip        heap.percent ram.percent cpu load_1m load_5m load_15m node.role master name
127.0.0.1           26          67   7    1.23    1.27     1.22 mdi       *      seabra

GET /_cat/indices?v :: Informa os índices disponíveis

health status index   uuid                   pri rep docs.count docs.deleted store.size pri.store.size
green  open   .kibana  6jook1rhQ9KIJrbBezTbDw   1   0          1            0        4kb            4kb
yellow open   customer odLb1nwjQtKeqIcfuoeOlA   5   1          0            0      1.1kb          1.1kb

Health = yellow :: yellow means that some replicas are not (yet) allocated. The reason this happens for this index is because Elasticsearch by default created one replica for this index. Since we only have one node running at the moment, that one replica cannot yet be allocated
